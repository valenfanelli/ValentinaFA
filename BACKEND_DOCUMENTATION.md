# Backend Documentation вЂ” Spring Boot + PostgreSQL

## Overview

This document describes the Java Spring Boot backend built for the portfolio application. The backend provides a REST API with bilingual support (English/Spanish) for **Education** and **Project** data, backed by a PostgreSQL database.

> **Note:** The production frontend reads data from static JSON files (i18n translation files) and does not require the backend to run. The backend is available on the `backend` branch as a reference implementation and can be deployed independently if needed.

## Database Schema

### Tables (auto-generated by Hibernate)

#### `educations`
| Column | Type | Description |
|--------|------|-------------|
| id | BIGSERIAL PK | Auto-generated ID |
| start_year | INTEGER | Year started |
| end_year | INTEGER | Year ended |

#### `education_translations`
| Column | Type | Description |
|--------|------|-------------|
| id | BIGSERIAL PK | Auto-generated ID |
| education_id | BIGINT FK | References `educations.id` |
| locale | VARCHAR(5) | Language code (`en`, `es`) |
| institution | VARCHAR(255) | Institution name |
| degree | VARCHAR(255) | Degree/course name |
| description | VARCHAR(2000) | Additional details |

Unique constraint: `(education_id, locale)`

#### `projects`
| Column | Type | Description |
|--------|------|-------------|
| id | BIGSERIAL PK | Auto-generated ID |
| url | VARCHAR(255) | Project URL |

#### `project_translations`
| Column | Type | Description |
|--------|------|-------------|
| id | BIGSERIAL PK | Auto-generated ID |
| project_id | BIGINT FK | References `projects.id` |
| locale | VARCHAR(5) | Language code (`en`, `es`) |
| title | VARCHAR(255) | Project title |
| description | VARCHAR(2000) | Project description |

Unique constraint: `(project_id, locale)`

## Database Setup

```sql
CREATE DATABASE portfolio_db;
CREATE USER portfolio_user WITH ENCRYPTED PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE portfolio_db TO portfolio_user;
GRANT ALL PRIVILEGES ON SCHEMA public TO portfolio_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO portfolio_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO portfolio_user;
```

Tables are created automatically by Hibernate on startup (`ddl-auto=update`).

## Configuration

### `application.properties`

```properties
spring.datasource.url=${DB_URL:jdbc:postgresql://localhost:5432/portfolio_db}
spring.datasource.username=${DB_USERNAME:portfolio_user}
spring.datasource.password=${DB_PASSWORD:root}
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

Supports environment variables for deployment flexibility.

## API Endpoints

### Locale negotiation
1. `?lang=es` query parameter (highest priority)
2. `Accept-Language: es` header
3. Defaults to `en`

### Education endpoints

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/api/educations` | List all educations |
| GET | `/api/educations/{id}` | Get single education |
| POST | `/api/educations` | Create education with translations |
| PUT | `/api/educations/{id}` | Update education |
| DELETE | `/api/educations/{id}` | Delete education |

### Project endpoints

| Method | URL | Description |
|--------|-----|-------------|
| GET | `/api/projects` | List all projects |
| GET | `/api/projects/{id}` | Get single project |
| POST | `/api/projects` | Create project with translations |
| PUT | `/api/projects/{id}` | Update project |
| DELETE | `/api/projects/{id}` | Delete project |

### Example: Create bilingual education

```bash
curl -X POST -H "Content-Type: application/json" -d '{
  "startYear": 2016,
  "endYear": 2024,
  "translations": [
    { "locale": "en", "institution": "National University of La Plata", "degree": "Computer Engineering", "description": "Average 8.2" },
    { "locale": "es", "institution": "Universidad Nacional de La Plata", "degree": "Ingeniera en computaciГіn", "description": "Promedio 8.2" }
  ]
}' http://localhost:8080/api/educations
```

### Example: Get projects in Spanish

```bash
curl "http://localhost:8080/api/projects?lang=es"
```

### Prerequisites
- Java 17+

mvn spring-boot:run

```bash

```

## Key Design Decisions

- **One-to-many translations**: Each entity has a separate translations table, allowing any number of languages without schema changes
- **Translation fallback**: Requested locale в†’ `en` в†’ any available translation
- **CORS**: Enabled for all origins (`@CrossOrigin(origins = "*")`)
- **Optimized queries**: `LEFT JOIN FETCH` to avoid N+1 query problems
- **CRUD support**: Full create/read/update/delete with translation upsert on update
